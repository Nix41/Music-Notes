
Crash Course

Piensa en las ramas como las ramas de un arbol
El tronco es la rama que se llama develop
El troco no se modifica como tal, se saca siempre de el una rama que se modifica, y cuando 
este lista , se "une", y las modificaciones de esta ahora pasan al tronco principal

Para descargar el proyecto, vas a git y descargas el zip, y lo abres
Cada ves que quieras actualizar el proyecto xk yo hice algun cambio y lo subi:
    1- Te paras en develop: git checkout develop
    2- git pull

Para traer otras ramas que no sean develop:
    - git fetch

Todo el tiempo estas parado en alguna rama

Para crear una rama:
Las ramas no tienen espacios en su nombre:
    - git checkout -b tu-nombre_razon_la_creaion_de_la_rama
    Esto crea una rama nueva que es una copia de la que estas parado
Siempre crea nuevas ramas parado en develop

Para moverte de una rama a otra:
    git checkout rama_a_la_que_te_quieres_mover

Para saber en que rama estas: git checkout
Para saber que ramas exiten: git branch

Cada rama que creas , se encuentra en tu pc local, 
para subirla al repo git y que yo pueda acceder a ella tienes que hacer esto la primera vez:
    git push --set-upstream origin tu_rama (esto conectado a internet)

Cada cambio que quieras guardar y que yo lo vea, haces:
    1- git add . (con el punto inluido)
    2- git commit -m 'mensaje de este commit, que es lo que estas guardando'
    3- git push (esto conecado a internet)

Si solo quieres guardar el cambio, y no lo quieres subir todavia, solo haces los primeros dos pasos
